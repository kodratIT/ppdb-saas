## Best Practices for Using These Templates

### 1. Always Include Context
- Task ID from task files
- Current phase of development
- Dependencies on other tasks

### 2. Be Specific
- Don't say "fix the bug"
- Say "fix the bug where users can't submit the registration form"

### 3. Include Acceptance Criteria
- Clear definition of done
- Testable conditions
- Specific outcomes

### 4. Reference Documentation
- Refer to .context.md for conventions
- Refer to docs/ for requirements
- Refer to task files for details

### 5. Define Output Expected
- What exactly do you want AI to produce?
- Code? Documentation? Instructions?
- Testing steps?

### 6. Use Consistent Format
- Always use the same structure
- This helps AI understand your requests better
- Reduces misunderstandings

### 7. Provide Relevant Code
- Paste relevant snippets when debugging
- Don't paste entire files unless necessary
- Focus on the problematic section

### 8. Include Environment Details
- OS, Node version, browser
- This helps with debugging
- Especially for environment-specific issues

### 9. Update Task Status
- Mark tasks as DONE when complete
- Update blockers if any
- Note any deviations from original plan

### 10. Verify Before Moving On
- Test locally
- Run lint/type-check
- Ensure all acceptance criteria met
